# git 基本操作

## 状態確認

```bash
# 現在の状態を確認
git status

# 変更差分を確認（ステージング前）
git diff

# コミット履歴を確認
git log

# コミット履歴を1行で確認
git log --oneline
```

---

## ファイルをステージングする

```bash
# 特定のファイルをステージング
git add ファイルパス

# 複数ファイルをステージング
git add ファイルパス1 ファイルパス2

# 変更されたファイルをすべてステージング
git add -A
```

---

## コミットする

```bash
# メッセージ付きでコミット
git commit -m "コミットメッセージ"
```

---

## ブランチ操作

```bash
# ブランチ一覧を確認
git branch

# ブランチを作成
git branch ブランチ名

# ブランチを切り替え
git checkout ブランチ名

# ブランチを作成して切り替え（上2つをまとめて）
git checkout -b ブランチ名

# ブランチを削除
git branch -d ブランチ名
```

---

## リモートとのやり取り

```bash
# リモートの変更を取得してマージ
git pull

# リモートにプッシュ
git push

# 初回プッシュ（リモートブランチを作成）
git push -u origin ブランチ名
```

---

## 変更ファイルの確認

### 現時点（未コミット）の変更ファイル一覧

```bash
git status
```

### 現時点（未コミット）のファイルごとの差分

```bash
# すべての変更ファイルの差分
git diff

# 特定ファイルの差分
git diff ファイルパス

# ステージング済みの差分（git add 後）
git diff --staged
```

### 指定コミットの変更ファイル一覧

```bash
# コミットハッシュはgit log --oneline で確認する
git show --stat コミットハッシュ

# 例
git show --stat a1b2c3d
```

### 指定コミットのファイルごとの差分

```bash
# コミット全体の差分
git show コミットハッシュ

# 特定ファイルのみの差分
git show コミットハッシュ -- ファイルパス

# 例
git show a1b2c3d -- src/pages/index.astro
```

---

## 変更を元に戻す

```bash
# ステージングを取り消す（ファイルの変更は残る）
git restore --staged ファイルパス

# ファイルの変更を取り消す（変更が消えるので注意）
git restore ファイルパス
```

## よく使うコミットメッセージの書き方

| 種別 | 用途 | 例 |
|---|---|---|
| `feat` | 新機能追加 | `feat: トップページを追加` |
| `fix` | バグ修正 | `fix: ヘッダーの表示崩れを修正` |
| `style` | スタイル変更 | `style: CSS の順序を修正` |
| `refactor` | リファクタリング | `refactor: コンポーネントを分割` |
| `docs` | ドキュメント | `docs: README を更新` |
| `chore` | 設定・ツール変更 | `chore: stylelint を導入` |
