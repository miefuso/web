# CSS 命名規約

## 基本方針

**FLOCSS + BEM** をベースにした命名規約を採用。

---

## プレフィックス一覧

| プレフィックス | カテゴリ  | 役割                         | 例                              |
| -------------- | --------- | ---------------------------- | ------------------------------- |
| `l-`           | Layout    | ページ全体の構造・骨格       | `l-header`、`l-footer`          |
| `p-`           | Project   | ページ固有のセクション       | `p-fv`、`p-feature`、`p-access` |
| `c-`           | Component | 複数箇所で使い回す汎用パーツ | `c-btn`、`c-card`               |

---

## BEM 記法

ブロック内の要素は `__`（アンダースコア2つ）で繋ぐ。

```
ブロック
ブロック__要素
ブロック__要素--修飾子
```

### 例

```scss
.l-header {
} // ブロック
.l-header__inner {
} // ブロック内の要素
.l-header__nav {
} // ブロック内の要素
.l-header__nav-list {
} // ブロック内の要素（複数単語はハイフン区切り）

.c-btn {
} // ブロック
.c-btn--primary {
} // 修飾子（バリエーション）
.c-btn--outline {
} // 修飾子（バリエーション）
```

## ディレクトリと対応関係

```
src/assets/styles/
├── foundation/           // プレフィックスなし（body、.container など）
│   ├── _reset.scss
│   ├── _base.scss
│   └── _variables.scss
├── layout/               // l- プレフィックス
│   ├── _header.scss
│   └── _footer.scss
├── object/
│   └── component/        // c- プレフィックス
│       └── _button.scss
└── page/                 // p- プレフィックス
    └── _top.scss
```

---

## 命名ルール補足

- 単語の区切りはハイフン `-`（例：`p-feature__nav-list`）
- キャメルケース・アンダースコア1つは使わない
- ブロック名はページ・コンポーネントの役割を表す英単語にする
