## 作業概要

Astro の基本的な仕様を学習しながらコーポレートサイトのプロジェクト基盤を整備した。

---

## プロジェクト構成

```
src/
├── assets/
│   ├── images/   ← <Image> で最適化対象（WebP変換など）
│   ├── styles/   ← SCSS ファイル
│   └── scripts/  ← TypeScript ファイル
├── components/
│   ├── common/   ← Header・Footer など全ページ共通
│   ├── ui/       ← Button など汎用パーツ
│   └── icons/    ← アイコン関連
├── layouts/      ← BaseLayout など共通レイアウト
├── pages/        ← ファイル名がそのまま URL になる
└── utils/        ← 汎用関数・定数（seo.ts など）
```

```
public/
└── images/       ← CSS background-image 用（最適化なし）
```

---

## 画像管理

| 配置場所             | 用途                               | ビルド後                                    |
| -------------------- | ---------------------------------- | ------------------------------------------- |
| `src/assets/images/` | `<Image>` コンポーネントで使う画像 | WebP 変換・ハッシュ付きで `dist/_astro/` へ |
| `public/images/`     | CSS background-image など          | そのまま `dist/images/` へコピー            |

---

## ルーティング

- `src/pages/index.astro` → `/`
- `src/pages/about.astro` または `src/pages/about/index.astro` → `/about/`
- `_` で始まるファイルはページとして出力されない
- デフォルトは `directory` 形式（`about/index.html`）で末尾スラッシュあり URL

---

## ビルド

```bash
npm run dev      # 開発サーバー起動（localhost:4321）
npm run build    # dist/ に静的ファイルを書き出し
npm run preview  # ビルド結果をローカルで確認
```

### dist/ の出力

- CSS はソース量が少ない場合は HTML に `<style>` としてインライン化、増えると `dist/_astro/*.css` に外部ファイル化
- 画像（`src/assets/` 配下）は WebP 変換・ハッシュ付きファイル名で `dist/_astro/` へ
- `public/` 配下はそのまま `dist/` へコピー
